Animasyon Felsefesi — Nasıl hareket etmeli?

Mevcut sitede CSS keyframe animasyonları var (fadeInUp, glow-pulse, scroll-hint). "Pro Max" seviyeye taşımak için hangi hareket dili doğru?

A

Subtle & Refined — "Neredeyse fark edilmez ama hissedilir"

Mevcut CSS animasyonlarını ince ayarla. Scroll reveal için IntersectionObserver ekle, buton hover'larında cubic-bezier(0.16,1,0.3,1) spring easing. Framer Motion yok — bundle boyutu küçük kalır. Sayfa hızlı, animasyonlar %10 daha iyi hissettiren "ghost" dokunuşlar.

Artılar

  • Sıfır yeni bağımlılık
  • Hızlı yükleme
  • prefers-reduced-motion zaten var
  • 2-3 saat iş

Eksiler

  • Karmaşık sekanslar zor
  • "Wow" anı yok
B

Framer Motion — Sadece kritik noktalarda

Framer Motion eklenir ama seçici kullanılır: Hero text stagger, Bento Grid kartları için whileInView, Team Section için layout animasyonu. Geri kalan her şey CSS kalır. ~31KB gzip ek yük — Next.js code splitting ile sadece ihtiyaç anında yüklenir.

Artılar

  • Spring physics — gerçekçi hareket
  • Stagger sekansları kolaylaşır
  • Gesture desteği (hover, drag)
  • Orta maliyet

Eksiler

  • 31KB ek bundle
  • Dikkatli kullanım gerekir
C

CSS Only + View Transitions API — Sıfır bağımlılık, maksimum modern

Framer Motion yok. Bunun yerine: CSS @keyframes genişletilir, animation-timeline: scroll() ile scroll-driven animasyonlar, Next.js 15'in desteklediği View Transitions API ile sayfa geçişleri. En modern yaklaşım — ama tarayıcı desteği henüz %78.

Artılar

  • Sıfır JS yükü
  • Cutting-edge his
  • GPU hızlandırmalı

Eksiler

  • Safari'de kısmi destek
  • Karmaşık sekanslar sınırlı
  • Daha az kontrol

Önerim

B — Framer Motion seçici kullanım. Sinematik Hero + Terminal Roster + Dergi Grid seçtiniz — bu üçünde de stagger ve spring animasyonlar Framer Motion olmadan doğal hissetmez. CSS ile yapılabilir ama 3× daha fazla kod gerekir ve sonuç mekanik görünür. Next.js dynamic import ile sadece ihtiyaç duyulan component'larda yüklenir.
